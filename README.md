PART A – WEBSITE PLANNING & DOCUMENTATION

1. Website Specifications



a. Website Objective and Goals

The website being developed is for a gym called Wizzys Gym. The main objective of the website is to promote the gym and make it easier for people to learn about its services and join.

The goals of the website are:

To attract new members by showing what the gym offers

To provide clear information about services such as personal training and group classes

To display membership plans and pricing

To allow users to contact the gym easily

To build a strong online presence for the business

This helps the gym grow by reaching more people and making its services more accessible.

b. Target Audience

The target audience for this website includes people who are interested in fitness and improving their health.

Demographics:

Age: 16–40 years old

Gender: Male and female

Location: Mainly people living in urban areas (e.g. Gaborone)

Lifestyle: Students, working individuals, and fitness enthusiasts

Technical Skills: Most users are familiar with using smartphones and the internet. They can browse websites, click links, and fill in forms without difficulty.

Because of this, the website should:

Be simple and easy to use

Work well on mobile devices

Load quickly without delays

c. Type of Website

This website is mainly a business and informational website.

It will:

Promote the gym and its services

Provide information such as pricing and schedules

Include interactive features like contact and feedback forms

The website is designed to both inform users and encourage them to join the gym.

2. Information Design & Taxonomy



The website will be organized in a clear and simple structure so that users can find information quickly without getting confused.

Site Structure (Site Map)

Home

About Us

Services

Personal Training

Group Classes

Membership Plans

Contact / Feedback

Navigation

The Home page will act as the main starting point of the website.

All other pages will be accessible through a navigation bar at the top of the page.

Each page will include links that take the user back to the Home page.

Users should be able to find any important information within 2–3 clicks.

This makes the website easier to use and prevents users from getting lost while browsing.

Content Organization (Taxonomy)

The content is grouped into clear categories so users can easily understand where to find things:

Information pages → About Us

Service-related content → Services

Business offerings → Membership Plans

User interaction → Contact/Feedback

This structure keeps everything organized and improves the overall user experience.

3. Page Template Design



The website will follow the same layout across all pages so that it looks consistent and professional.

Page Layout

Header

Gym name/logo

Navigation menu (Home, About, Services, Plans, Contact)

Main Content Area

Banner or heading at the top

Page-specific content such as text and images

Footer

Contact details

Social media links

The layout will be simple and clean, with enough spacing so that the content is easy to read and not overcrowded.

Wireframe Description

Each page will follow the same basic structure:

Header (Logo + Navigation)

Main Section

Title

Image or banner

Content (text/images)


Footer (Contact + Socials)

This keeps the design consistent across all pages and makes the website easier to use.

4. Technical Brief



The website will be developed using the following technologies:

HTML5 – used to create the structure of the web pages

CSS3 – used to style the website and control layout

JavaScript – used for basic interactivity such as form validation

Bootstrap – used to make the website responsive on different screen sizes

Tools and Resources:

Visual Studio Code (VS Code) – for writing and editing code

GitHub – for storing and managing the project files

GitHub Pages – for hosting the website online

GIMP or similar software – for editing images and creating graphics

These tools are suitable because they support modern web development and match the requirements of the project.
